### 🌟 自我介绍 | About Me

**中文：** 你好！我是 24kmgic，一名来自中国的大一网安学生。 我在了解和学习 Linux 之后，发现了许多乐趣。 我想要自己去了解关于一些底层内核的知识，又想创建一个自己的内核。所以我把两件事情一起做了—— whoseRustyOS 就这样诞生了。 这个想法萌生于 2026 年 3 月。期间我翻阅了大量资料，经历了“尝试 -> 放弃 -> 再次挑战”的轮回。最终，我还是回到了这里。

### 🤝 致同行者 | To Fellow Travelers

**中文：** 如果你像我一样，对编程有一定了解，但还没接触过 Rust 语言，请千万不要害怕：

- 别被语法吓到：只要能看懂逻辑，其实你离写出来就不远了。

- 善用 AI：遇到晦涩的语法，尽管去问 AI，它是好的老师。

- 推荐资源：我是跟着大佬的教程起步的，强烈推荐这篇适合小白入坑的博客： [Writing an OS in Rust](https://os.phil-opp.com/) (Philipp Oppermann)。 

  内核开发是一条孤独的路，但我希望能在这里遇到一起交流、一起秃头的伙伴。我想和你一起学习！

**English:** If you are like me—having some understanding of programming but haven't touched the Rust language yet—please don't be afraid:

- Don't be scared by the syntax: As long as you can understand the logic, you're not far from being able to write it.

- Make good use of AI: When encountering obscure syntax, feel free to ask AI; it is an excellent mentor.

- Recommended Resource: I started by following a great tutorial. I highly recommend this blog for beginners: [Writing an OS in Rust](https://os.phil-opp.com/) (Philipp Oppermann). 

  Kernel development is a lonely path, but I hope to meet partners here to exchange ideas and perhaps go bald together. I want to learn with you!

### 📁 仓库结构 | Repository Structure

**中文：**

- **src/ (Code)**：存放我的核心代码（在 Docker 环境中维护）。
- **notes/ (Study Notes)**：我的学习笔记。你可以把它当成知识点总结，它会像“导读”一样帮助你更轻松地阅读原版博客。原版我看着看着就会有不懂的东西，我把它们记录下来，省去你们查找资料的时间啦。

**English:**

- **src/ (Code)**: Stores my core code (maintained in a Docker environment).
- **notes/ (Study Notes)**: My study notes. You can treat them as a summary of knowledge points to help you read the original blog more easily. I record the things I find confusing while reading the original version to save you time on searching for information.
